# 游戏商城数据库设计文档

## 📊 数据库概览

**数据库名称**: `gacha_system`  
**总表数**: 20 张表  
**字符集**: utf8mb4  
**存储引擎**: InnoDB

## 📋 表结构清单

### 核心业务表

#### 1. 用户相关 (6 张表)

| 表名 | 说明 | 主要功能 |
|------|------|----------|
| `users` | 用户表 | 用户基本信息、支持手机/短信/二维码登录 |
| `user_wallets` | 用户钱包表 | 管理用户余额、冻结资金、累计充值/消费 |
| `wallet_transactions` | 钱包流水表 | 记录所有资金变动（充值/购买/退款/提现） |
| `login_logs` | 登录日志表 | 安全审计、记录登录行为 |
| `admins` | 管理员表 | 后台管理人员 |
| `admin_operation_logs` | 管理员操作日志 | 审计管理员操作 |

#### 2. 游戏相关 (5 张表)

| 表名 | 说明 | 主要功能 |
|------|------|----------|
| `games` | 游戏表 | 游戏基本信息、价格、折扣、销量、评分 |
| `game_categories` | 游戏分类表 | 支持多级分类（动作/冒险/赛车等） |
| `game_tags` | 游戏标签表 | 游戏标签（开放世界/犯罪/多人等） |
| `game_category_mapping` | 游戏分类关联表 | 游戏与分类的多对多关系 |
| `game_tag_mapping` | 游戏标签关联表 | 游戏与标签的多对多关系 |
| `game_system_requirements` | 游戏配置要求表 | 最低/推荐配置（CPU/内存/显卡等） |

#### 3. 购物相关 (3 张表)

| 表名 | 说明 | 主要功能 |
|------|------|----------|
| `shopping_cart` | 购物车表 | 用户购物车商品 |
| `wishlists` | 愿望单表 | 收藏游戏、折扣通知设置 |
| `orders` | 订单表 | 订单主表（金额/支付状态/订单状态） |
| `order_items` | 订单详情表 | 订单中的具体游戏商品 |

#### 4. 互动相关 (3 张表)

| 表名 | 说明 | 主要功能 |
|------|------|----------|
| `game_reviews` | 游戏评论表 | 用户评论、评分、回复 |
| `review_images` | 评论图片表 | 评论中的截图 |
| `system_notifications` | 系统通知表 | 折扣通知、发售通知、订单通知等 |

#### 5. 运营相关 (1 张表)

| 表名 | 说明 | 主要功能 |
|------|------|----------|
| `banners` | 轮播图表 | 首页轮播广告、活动推广 |

## 🔍 核心功能表结构详解

### 1. users - 用户表

```sql
核心字段:
- id: 主键
- username: 用户名（唯一）
- password_hash: 密码哈希
- phone: 手机号（支持短信登录）
- phone_verified: 手机号验证状态
- email: 邮箱
- email_verified: 邮箱验证状态
- avatar_url: 头像 URL
- nickname: 昵称
- signature: 个性签名
- account_status: 账号状态（0-封禁 1-正常 2-冻结）
- user_level: 用户等级
- experience_points: 经验值
- last_login_time: 最后登录时间
- last_login_ip: 最后登录 IP
- login_type: 登录方式（password/sms/qr）
```

**特点**:
- ✅ 支持多种登录方式（密码/短信/二维码）
- ✅ 手机号和邮箱验证机制
- ✅ 账号状态管理
- ✅ 用户等级和经验值系统

### 2. user_wallets - 用户钱包表

```sql
核心字段:
- user_id: 用户 ID（与 users 一对一）
- balance: 可用余额
- frozen_balance: 冻结余额
- total_recharge: 累计充值
- total_consumed: 累计消费
```

**特点**:
- ✅ 独立管理用户资金
- ✅ 支持冻结余额（退款中/争议订单）
- ✅ 统计充值和消费记录

### 3. games - 游戏表

```sql
核心字段:
- title: 游戏标题
- short_description: 简短描述
- full_description: 完整描述
- cover_image: 封面图
- banner_image: 横幅图
- screenshots: 截图（JSON 数组）
- trailer_url: 预告片 URL
- base_price: 基础价格
- current_price: 当前价格
- discount_rate: 折扣率
- discount_start/end: 折扣时间
- is_featured: 是否精选
- is_on_sale: 是否在售
- rating: 评分（0-10）
- total_sales: 总销量
- download_count: 下载量
- file_size: 文件大小
```

**特点**:
- ✅ 完整的商品信息
- ✅ 折扣系统（支持限时折扣）
- ✅ 评分和销量统计
- ✅ 多媒体支持（截图/预告片）

### 4. game_system_requirements - 配置要求表

```sql
核心字段:
- os_min/recommended: 操作系统
- cpu_min/recommended: CPU
- ram_min/recommended: 内存
- gpu_min/recommended: 显卡
- directx_min/recommended: DirectX
- storage_min/recommended: 存储空间
- network_min/recommended: 网络要求
- sound_card_min/recommended: 声卡
- additional_notes: 额外说明
```

**特点**:
- ✅ 详细的最低和推荐配置
- ✅ 覆盖所有硬件要求
- ✅ 与游戏表一对一关系

### 5. wishlists - 愿望单表

```sql
核心字段:
- user_id: 用户 ID
- game_id: 游戏 ID
- priority: 优先级（1-普通 2-重要 3-非常想要）
- notify_discount: 是否通知折扣
- notify_release: 是否通知发售
```

**特点**:
- ✅ 支持优先级设置
- ✅ 可配置折扣通知
- ✅ 自动通知机制（需配合后端定时任务）

### 6. orders - 订单表

```sql
核心字段:
- order_no: 订单编号（唯一）
- user_id: 用户 ID
- total_amount: 订单总金额
- discount_amount: 优惠金额
- actual_amount: 实际支付金额
- payment_method: 支付方式（balance/alipay/wechat）
- payment_status: 支付状态（pending/paid/failed/refunded）
- order_status: 订单状态（pending/completed/cancelled）
- payment_time: 支付时间
- refund_time: 退款时间
- refund_reason: 退款原因
```

**特点**:
- ✅ 支持多种支付方式（余额/支付宝/微信）
- ✅ 完整的订单状态流转
- ✅ 退款支持

### 7. game_reviews - 评论表

```sql
核心字段:
- user_id: 用户 ID
- username: 用户名（快照）
- user_avatar: 用户头像（快照）
- game_id: 游戏 ID
- rating: 评分（1-10）
- title: 评论标题
- content: 评论内容
- pros: 优点
- cons: 缺点
- play_hours: 游戏时长
- is_verified_purchase: 是否已验证购买
- helpful_count: 有用数量
- parent_id: 父评论 ID（支持回复）
- is_official: 是否官方回复
- status: 状态（0-待审核 1-显示 2-隐藏）
```

**特点**:
- ✅ 支持评分（1-10 分）
- ✅ 优缺点分析
- ✅ 游戏时长展示
- ✅ 购买验证标记
- ✅ 评论回复（嵌套结构）
- ✅ 官方回复标记
- ✅ 评论审核机制

### 8. system_notifications - 通知表

```sql
核心字段:
- user_id: 用户 ID
- title: 通知标题
- content: 通知内容
- type: 类型（discount/release/system/order）
- is_read: 是否已读
- related_game_id: 关联游戏 ID
- related_order_id: 关联订单 ID
```

**特点**:
- ✅ 多种通知类型
- ✅ 支持关联游戏和订单
- ✅ 已读/未读状态

## 🔐 安全与审计

### login_logs - 登录日志
- 记录所有登录尝试
- 包含 IP 地址、设备信息、登录地点
- 用于安全分析和异常检测

### admin_operation_logs - 管理员操作日志
- 记录所有后台操作
- 包含操作前后的数据快照
- 支持责任追溯

## 💰 支付流程

```
1. 用户下单 → 创建订单（orders）
2. 选择支付方式：
   - 余额支付：扣除 user_wallets.balance
   - 二维码支付：支付宝/微信扫码
3. 支付成功 → 更新订单状态
4. 记录流水 → wallet_transactions
5. 更新游戏销量 → games.total_sales
6. 增加用户经验 → users.experience_points
```

## 📊 折扣通知流程

```
1. 用户添加游戏到愿望单（wishlists）
2. 设置 notify_discount = 1
3. 游戏打折时：
   - 检测 wishlists 表
   - 创建 system_notifications
   - 可选：发送邮件/短信
```

## 🎯 特色功能

### 1. 用户等级系统
- 通过 `experience_points` 字段记录经验值
- 购买游戏、评论等可获得经验
- 不同等级享受不同折扣

### 2. 评论验证系统
- `is_verified_purchase` 标记是否已购买
- 只有购买过的用户才能发表已验证评论
- 增加评论可信度

### 3. 愿望单优先级
- 优先级 1-3 标记重要程度
- 高优先级游戏打折时优先通知

### 4. 游戏配置要求
- 详细的最低/推荐配置
- 帮助用户判断是否能运行游戏

### 5. 资金安全
- 独立的钱包表管理资金
- 完整的流水记录
- 支持冻结余额（争议处理）

## 🔧 使用建议

1. **密码加密**: 使用 BCrypt 或 Argon2 算法
2. **短信验证**: 集成阿里云/腾讯云短信服务
3. **二维码登录**: 生成临时 token，轮询登录状态
4. **余额支付**: 使用数据库事务确保一致性
5. **折扣通知**: 定时任务扫描 wishlists 表
6. **图片存储**: 使用 OSS/CDN 存储图片
7. **评论审核**: 敏感词过滤 + 人工审核
8. **数据备份**: 定期备份数据库
