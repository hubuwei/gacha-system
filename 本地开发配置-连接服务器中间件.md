# 本地开发环境配置指南 - 连接服务器中间件

## 📋 概述

本指南教你如何在本地开发时，直接连接服务器上的Redis、MySQL、RabbitMQ等中间件，无需在本地安装和启动这些服务。

## ⚠️ 注意事项

1. **需要网络连接**：确保你的电脑能访问服务器 `111.228.12.167`
2. **防火墙设置**：服务器已开放相关端口（3306, 6379, 5672, 9200）
3. **数据隔离**：建议使用生产数据库 `gacha_system_prod`，注意不要误删数据
4. **性能考虑**：网络延迟可能比本地稍高，但在可接受范围内

## 🔧 配置步骤

### 方法一：使用 IDEA 环境变量配置（推荐）

#### 1. 复制配置文件

```powershell
# 在项目根目录执行
copy .env.local.example .env.local
```

#### 2. 配置 IDEA Run Configuration

**对于 mall-service：**

1. 打开 IDEA，点击右上角的运行配置下拉菜单
2. 选择 "Edit Configurations..."
3. 找到 mall-service 的配置（或创建新的 Spring Boot 配置）
4. 在 "Environment variables" 字段中添加：

```
DB_HOST=111.228.12.167;DB_PORT=3306;DB_NAME=gacha_system_prod;DB_USERNAME=root;DB_PASSWORD=Xc037417!
```

5. 点击 "Apply" 和 "OK"

**对于 auth-service：**

auth-service 已经支持环境变量，配置方式相同：

```
DB_HOST=111.228.12.167;DB_PORT=3306;DB_NAME=gacha_system_prod;DB_USERNAME=root;DB_PASSWORD=Xc037417!;REDIS_HOST=111.228.12.167;REDIS_PORT=6379;REDIS_PASSWORD=Xc037417!
```

#### 3. 修改 application.yml 启用 Redis/RabbitMQ

如果你需要使用 Redis 或 RabbitMQ，需要修改对应的配置文件：

**mall-service/src/main/resources/application.yml：**

```yaml
spring:
  redis:
    enabled: true  # 改为 true
    host: ${REDIS_HOST:localhost}
    port: ${REDIS_PORT:6379}
    password: ${REDIS_PASSWORD:}
  
  rabbitmq:
    enabled: true  # 改为 true
    host: ${RABBITMQ_HOST:localhost}
    port: ${RABBITMQ_PORT:5672}
    username: ${RABBITMQ_USERNAME:guest}
    password: ${RABBITMQ_PASSWORD:guest}
```

### 方法二：使用 VM Options（备选方案）

在 IDEA 的 Run Configuration 中，添加 VM options：

```
-DDB_HOST=111.228.12.167 -DDB_PORT=3306 -DDB_NAME=gacha_system_prod -DDB_USERNAME=root -DDB_PASSWORD=Xc037417!
```

### 方法三：直接修改 application.yml（最简单但不推荐）

直接修改 `mall-service/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://111.228.12.167:3306/gacha_system_prod?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: "Xc037417!"
```

⚠️ **注意**：这种方式会提交到 Git，建议使用 `.gitignore` 忽略或使用环境变量。

## 🚀 快速启动命令

### 启动 mall-service（连接服务器数据库）

```powershell
# 方式1：使用环境变量
$env:DB_HOST="111.228.12.167"
$env:DB_PORT="3306"
$env:DB_NAME="gacha_system_prod"
$env:DB_USERNAME="root"
$env:DB_PASSWORD="Xc037417!"
cd mall-service
mvn spring-boot:run

# 方式2：一行命令
$env:DB_HOST="111.228.12.167"; $env:DB_PASSWORD="Xc037417!"; cd mall-service; mvn spring-boot:run
```

### 启动 auth-service（连接服务器数据库和Redis）

```powershell
$env:DB_HOST="111.228.12.167"
$env:DB_PASSWORD="Xc037417!"
$env:REDIS_HOST="111.228.12.167"
$env:REDIS_PASSWORD="Xc037417!"
cd auth-service
mvn spring-boot:run
```

## ✅ 验证配置

启动服务后，检查控制台输出：

1. **数据库连接成功**：看到类似 "HikariPool-1 - Start completed" 的日志
2. **Redis连接成功**（如果启用）：没有连接错误
3. **应用启动成功**：看到 "Started XxxApplication in x.xxx seconds"

## 🔍 常见问题

### 1. 连接超时

**问题**：`Communications link failure` 或 `Connection timed out`

**解决**：
- 检查网络连接：`ping 111.228.12.167`
- 检查防火墙是否允许访问
- 确认服务器MySQL允许远程连接

### 2. 认证失败

**问题**：`Access denied for user 'root'@'xxx'`

**解决**：
- 检查密码是否正确
- 确认MySQL用户有远程访问权限

### 3. Redis连接失败

**问题**：`Cannot get Jedis connection`

**解决**：
- 检查Redis密码是否正确
- 确认Redis允许远程连接（已在服务器上配置）

## 💡 最佳实践

1. **使用环境变量**：避免硬编码敏感信息
2. **区分环境**：开发用 `.env.local`，生产用 `.env.production`
3. **Git忽略**：确保 `.env.local` 在 `.gitignore` 中
4. **定期备份**：操作生产数据库前做好备份
