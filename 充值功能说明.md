## 后端实现

### 1. 充值接口
**路径**: `POST /api/recharge`

**请求头**:
```
Authorization: Bearer {token}
Content-Type: application/json
```

**请求体**:
```json
{
  "amount": 100
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "record": {
      "id": 1,
      "userId": 1,
      "amount": 100,
      "points": 1000,
      "createTime": "2026-03-28T20:48:22",
      "remark": "充值"
    },
    "balance": 100.00,
    "points": 1000
  }
}
```

### 2. 充值记录查询接口
**路径**: `GET /api/recharge/records`

**请求头**:
```
Authorization: Bearer {token}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": 1,
      "userId": 1,
      "amount": 100,
      "points": 1000,
      "createTime": "2026-03-28T20:48:22",
      "remark": "充值"
    }
  ]
}
```

### 3. 核心逻辑

#### RechargeController
- 从 Token 中提取用户 ID，确保安全性
- 调用 RechargeService 处理充值业务
- 返回充值记录和更新后的余额、积分

#### RechargeService
- 事务性处理充值流程
- 更新或创建用户钱包
- 计算积分（1 元 = 10 积分）
- 创建充值记录

- 实时显示当前余额

### 2. 充值流程
```javascript
// handleRecharge 函数
1. 检查登录状态
2. 发送 POST 请求到 /api/recharge
3. 携带 Authorization Header (Bearer Token)
4. 请求成功后重新加载用户信息
5. 显示成功提示并关闭弹窗
```

### 3. 关键代码
```javascript
const handleRecharge = async () => {
    if (!isLoggedIn.value) {
        message.error('请先登录');
        return;
    }
    
    rechargeLoading.value = true;
    try {
        const response = await fetch(`${API_BASE}/api/recharge`, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'Authorization': `Bearer ${token.value}`
            },
            body: JSON.stringify({
                amount: selectedRechargeAmount.value
            })
        });
        
        const data = await response.json();
        
        if (data.code === 200) {
            message.success(`充值成功！已到账 ￥${selectedRechargeAmount.value}，获得 ${Math.floor(selectedRechargeAmount.value * 10)} 积分`);
            closeRechargeModal();
            await loadUserInfo(); // 重新加载用户信息
        } else {
            message.error(data.message || '充值失败');
        }
    } catch (error) {
        console.error('充值错误:', error);
        message.error('充值失败，请稍后重试');
    } finally {
        rechargeLoading.value = false;
    }
};
```

## 数据库表结构

### wallets 表
```sql
CREATE TABLE wallets (
    user_id BIGINT NOT NULL COMMENT '用户 ID',
    balance DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '虚拟货币余额（用于抽奖）',
    points INT NOT NULL DEFAULT 0 COMMENT '积分（可用于兑换等）',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### recharge_records 表
```sql
CREATE TABLE recharge_records (
    id BIGINT NOT NULL AUTO_INCREMENT COMMENT '记录 ID',
    user_id BIGINT NOT NULL COMMENT '用户 ID',
    amount DOUBLE NOT NULL COMMENT '充值金额',
    points INT NOT NULL COMMENT '获得的积分',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '充值时间',
    remark VARCHAR(50) COMMENT '备注',
    PRIMARY KEY (id)
);
```

- 充值成功后，顶部会显示更新的余额和积分
- 余额增加充值的金额
- 积分增加充值金额 × 10

## 安全特性

1. **Token 认证**: 所有充值请求都需要携带有效的 JWT Token
2. **服务器端验证**: 从 Token 中提取用户 ID，防止前端伪造
3. **事务支持**: 充值过程使用 @Transactional 保证数据一致性
4. **输入验证**: 后端验证用户是否存在、金额是否有效
