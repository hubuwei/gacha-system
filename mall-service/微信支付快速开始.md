## 🚀 快速测试步骤

### 1. 更新 Maven 依赖

```bash
cd mall-service
mvn clean install
```

### 2. 配置微信支付参数

编辑 `mall-service/src/main/resources/application.yml`：

```yaml
wechat:
  pay:
    app-id: wx1234567890abcdef        # 替换为你的 APPID
    mch-id: 1234567890                  # 替换为你的商户号
    api-v3-key: your-api-v3-key-here    # 替换为你的 API V3 KEY
    private-key-path: classpath:cert/apiclient_key.pem
    serial-no: ABCDEF1234567890         # 替换为你的证书序列号
    notify-url: http://localhost:8081/api/payment/wechat/notify
```

> **注意**：本地测试时，`notify-url` 可以暂时使用 localhost，但真实环境必须是外网可访问的 HTTPS 地址。

### 3. 准备证书文件

在 `mall-service/src/main/resources/cert/` 目录下放置商户私钥文件：
```
mall-service/
└── src/
    └── main/
        └── resources/
            └── cert/
                └── apiclient_key.pem    # 从微信商户平台下载
```

### 4. 启动服务

```bash
cd mall-service
mvn spring-boot:run
```

### 5. 打开测试页面

在浏览器中打开：
```
file:///E:/CFDemo/gacha-system/mall-service/wechat-pay-test.html
```

或者将文件复制到前端项目的 public 目录，通过 HTTP 访问。

### 6. 测试流程

1. 填写用户 ID、游戏 ID、订单金额
2. 点击"创建订单并生成二维码"
3. 页面会显示支付二维码
4. 使用微信扫码支付（测试环境可使用模拟数据）
5. 系统会自动轮询查询支付状态

## 📡 API 接口说明

### 1. Native 下单

**接口**: `POST /api/payment/wechat/native/order`

**请求体**:
```json
{
  "orderNo": "ORD20260410123456",
  "amount": 99.00,
  "description": "游戏商城订单"
}
```

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "codeUrl": "weixin://wxpay/bizpayurl?pr=xxxxx",
    "orderNo": "ORD20260410123456",
    "amount": 99.00
  }
}
```

### 2. 查询订单

**接口**: `GET /api/payment/wechat/order/query?orderNo={orderNo}`

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "orderNo": "ORD20260410123456",
    "status": "SUCCESS"
  }
}
```

### 3. 关闭订单

**接口**: `POST /api/payment/wechat/order/close`

**请求体**:
```json
{
  "orderNo": "ORD20260410123456"
}
```

### 4. 支付回调

**接口**: `POST /api/payment/wechat/notify`

这是微信支付成功后自动调用的接口，用于更新订单状态。

## 🔧 前端集成示例

### 在 OrderCheckout.jsx 中使用

当前前端已经支持微信支付选项，当用户选择微信支付并提交订单时：

1. 后端会返回 `codeUrl`
2. 前端显示二维码模态框
3. 用户扫码支付
4. 前端轮询查询支付状态
5. 支付成功后跳转订单详情

### 修改建议

在 `game-mall/src/components/OrderCheckout.jsx` 中，当收到微信支付的 `codeUrl` 后：

```javascript
if (result.data.codeUrl) {
  // 显示二维码
  setShowQRCode(true);
  
  // 开始轮询支付状态
  const checkInterval = setInterval(async () => {
    const statusResponse = await fetch(
      `${MALL_API_BASE}/payment/wechat/order/query?orderNo=${result.data.orderNo}`
    );
    const statusResult = await statusResponse.json();
    
    if (statusResult.data.status === 'SUCCESS') {
      clearInterval(checkInterval);
      // 支付成功，跳转
      navigate(`/orders/${result.data.orderId}`);
    }
  }, 3000);
}
```

## ⚠️ 注意事项

### 开发环境

- 可以使用测试商户号或 Mock 数据
- `notify-url` 可以是 localhost
- 建议使用沙箱环境测试

### 生产环境

- 必须使用正式商户号
- `notify-url` 必须是外网可访问的 HTTPS 地址
- 必须验证回调签名
- 私钥文件不要提交到 Git

## 🐛 常见问题排查

### 问题 1: 启动时报错"找不到证书文件"

**解决**: 确保证书文件路径正确
```yaml
private-key-path: classpath:cert/apiclient_key.pem
```

检查文件是否存在：
```bash
ls mall-service/src/main/resources/cert/apiclient_key.pem
```

### 问题 2: 下单失败"签名错误"

**原因**: 配置参数不正确

**解决**: 检查以下配置是否准确
- `app-id`: 公众号/小程序 APPID
- `mch-id`: 商户号
- `api-v3-key`: APIv3 密钥
- `serial-no`: 证书序列号

### 问题 3: 收不到回调通知

**原因**: 回调地址不可访问

**解决**: 
1. 确保 `notify-url` 是外网可访问的地址
2. 使用内网穿透工具（ngrok/frp）
3. 检查防火墙设置

### 问题 4: 二维码无法扫描

**原因**: codeUrl 格式错误或已过期

**解决**:
1. 检查返回的 `codeUrl` 是否以 `weixin://` 开头
2. 二维码有效期为 2 小时，过期需重新生成
