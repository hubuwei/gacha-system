## 🔧 中间件使用说明

### 1. Redis 使用场景

- **缓存**: 游戏信息、用户信息、配置信息
- **Session 管理**: 用户登录状态
- **分布式锁**: 秒杀、库存扣减
- **计数器**: 游戏销量、评论数
- **消息队列**: 简单的异步任务

### 2. Elasticsearch 使用场景

- **游戏搜索**: 全文检索、多条件筛选
- **日志分析**: 操作日志、访问日志
- **数据分析**: 销售数据、用户行为

### 3. RabbitMQ 使用场景

- **异步处理**: 发送邮件/短信、生成报表
- **削峰填谷**: 秒杀活动、批量下单
- **系统解耦**: 订单系统→库存系统→支付系统
- **延迟队列**: 订单超时取消

## 📝 核心业务流程

### 1. 用户登录流程

```
1. 用户提交登录请求 → UserController.login()
2. 验证用户名密码 → UserService.authenticate()
3. 生成 JWT Token → JwtUtil.generateToken()
4. 缓存用户信息到 Redis → RedisUtil.setCacheObject()
5. 记录登录日志 → LoginLogService.create()
6. 返回 Token 和用户信息
```

### 2. 购买游戏流程

```
1. 用户提交订单 → OrderController.createOrder()
2. 检查库存和价格 → GameService.checkStock()
3. 创建订单（事务）→ OrderService.createOrder()
   - 创建订单主表
   - 创建订单详情
   - 扣减库存
   - 记录流水
4. 发送 MQ 消息 → OrderProducer.sendOrderCreated()
5. 消费者处理异步任务:
   - 更新游戏销量
   - 增加用户经验
   - 发送通知
6. 返回订单信息
```

### 3. 愿望单折扣通知流程

```
1. 定时任务扫描打折游戏 → DiscountScheduler.checkDiscounts()
2. 查询愿望单用户 → WishlistRepository.findByGameId()
3. 创建通知 → NotificationService.create()
4. 发送 MQ 消息 → NotificationProducer.send()
5. 消费者推送消息:
   - WebSocket 推送
   - 邮件通知
   - 短信通知
```

## 🚀 启动说明

### 前置条件

1. MySQL 8.0+ (已创建数据库 gacha_system)
2. Redis 6.0+
3. Elasticsearch 7.x
4. RabbitMQ 3.x
5. JDK 11+

### 启动步骤

```bash
# 1. 启动 MySQL
# 确保数据库已创建并导入表结构

# 2. 启动 Redis
redis-server

# 3. 启动 Elasticsearch
# (根据系统不同使用相应命令)

# 4. 启动 RabbitMQ
rabbitmq-server

# 5. 编译项目
cd mall-service
mvn clean package -DskipTests

# 6. 启动服务
java -jar target/mall-service-1.0.0-SNAPSHOT.jar

# 或直接运行
mvn spring-boot:run
```

## 📡 API 接口列表

### 用户模块
- POST /api/user/register - 用户注册
- POST /api/user/login - 用户登录
- POST /api/user/logout - 用户登出
- GET /api/user/info - 获取用户信息
- PUT /api/user/profile - 更新用户资料
- POST /api/user/avatar - 上传头像

### 游戏模块
- GET /api/games - 游戏列表
- GET /api/games/{id} - 游戏详情
- GET /api/games/search - 游戏搜索
- POST /api/games - 创建游戏（管理员）
- PUT /api/games/{id} - 更新游戏（管理员）

### 购物车模块
- GET /api/cart - 获取购物车
- POST /api/cart/items - 添加到购物车
- DELETE /api/cart/items/{id} - 删除购物车项
- PUT /api/cart/items/{id} - 更新购物车项

### 订单模块
- POST /api/orders - 创建订单
- GET /api/orders/{id} - 订单详情
- GET /api/orders/list - 订单列表
- POST /api/orders/{id}/pay - 支付订单
- POST /api/orders/{id}/cancel - 取消订单

### 支付模块
- POST /api/payment/wechat - 微信支付
- POST /api/payment/alipay - 支付宝支付
- POST /api/payment/balance - 余额支付

## 🔐 安全说明

1. 所有 API 都需要 JWT Token 认证（除了登录注册）
2. Token 放在 Header 中：Authorization: Bearer {token}
3. 管理员接口需要管理员权限
4. 敏感操作需要二次验证（支付、修改密码等）
