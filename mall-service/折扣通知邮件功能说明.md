# 折扣通知邮件功能使用指南

## 📋 功能概述

当管理员在后台调整游戏折扣时，系统会自动向将该游戏添加到愿望单并开启"折扣通知"的用户发送邮件提醒。

## 🔧 工作流程

```
1. 用户将游戏添加到愿望单 → 开启"折扣通知"选项
2. 管理员更新游戏折扣 → PUT /api/admin/games/{gameId}/discount
3. 系统检测折扣提升 → 触发邮件通知
4. 异步发送邮件 → 用户收到折扣提醒
5. 创建站内通知 → 用户可在通知中心查看
```

## 📝 API 接口

### 更新游戏折扣（管理员）

**接口地址：** `PUT /api/admin/games/{gameId}/discount`

**请求参数：**
```json
{
  "discountRate": 50,        // 折扣率 (0-100)
  "currentPrice": 149.00     // 折后价格
}
```

**响应示例：**
```json
{
  "code": 200,
  "message": "折扣更新成功，已通知相关用户",
  "data": null
}
```

**错误响应：**
```json
{
  "code": 400,
  "message": "折扣率必须在0-100之间",
  "data": null
}
```

## ✉️ 邮件配置

当前配置使用 **QQ邮箱 SMTP** 服务：

```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 587
    username: 1712133303@qq.com
    password: xwxqwrqawznddaij  # SMTP授权码
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
            required: true
```

### 如何获取SMTP授权码

1. 登录 QQ邮箱
2. 进入 **设置** → **账户**
3. 找到 **POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务**
4. 开启 **IMAP/SMTP服务**
5. 生成授权码（16位字符串）
6. 将授权码填入 `.env` 文件的 `MAIL_PASSWORD` 或直接修改 `application.yml`

### 使用其他邮箱服务商

#### 163邮箱

## 🎯 使用步骤

### 1. 用户侧操作

#### 添加游戏到愿望单并开启通知

**方式一：在游戏详情页**
```javascript
// 前端调用
POST /api/wishlist?userId={userId}&gameId={gameId}&notifyDiscount=true
```

**方式二：在愿望单侧边栏**
- 打开愿望单
- 勾选"折扣通知"复选框
- 系统自动保存设置

#### 关闭折扣通知

```javascript
// 前端调用
PUT /api/wishlist/{gameId}?userId={userId}&notifyDiscount=false
```

### 2. 管理员侧操作

#### 通过API更新折扣

**使用 curl：**
```bash
curl -X PUT http://localhost:8081/api/admin/games/1/discount \
  -H "Content-Type: application/json" \
  -d '{
    "discountRate": 50,
    "currentPrice": 149.00
  }'
```

**使用 Postman：**
1. 方法：`PUT`
2. URL：`http://localhost:8081/api/admin/games/1/discount`
3. Headers：`Content-Type: application/json`
4. Body（raw JSON）：
```json
{
  "discountRate": 50,
  "currentPrice": 149.00
}
```

#### 注意事项

- ✅ 只有当 **新折扣率 > 旧折扣率** 且 **新折扣率 > 0** 时才发送邮件
- ✅ 邮件是 **异步发送** 的，不会阻塞API响应
- ✅ 只会发送给 **开启了折扣通知** 的用户
- ✅ 如果用户没有绑定邮箱，会跳过该用户

### 常见问题排查

#### 1. 邮件发送失败

**检查项：**
- ✅ SMTP授权码是否正确
- ✅ 邮箱是否开启了SMTP服务
- ✅ 防火墙是否阻止了587端口
- ✅ 网络连接是否正常

**测试连接：**
```bash
telnet smtp.qq.com 587
```

#### 2. 用户未收到邮件

**检查项：**
- ✅ 用户是否绑定了邮箱
- ✅ 用户是否开启了"折扣通知"
- ✅ 垃圾邮件文件夹
- ✅ 邮件服务器延迟

**查询用户愿望单设置：**
```sql
SELECT w.*, u.email 
FROM wishlists w
JOIN users u ON w.user_id = u.id
WHERE w.game_id = 1 AND w.notify_discount = 1;
```

#### 3. 折扣更新但未触发邮件

**可能原因：**
- ❌ 新折扣率 ≤ 旧折扣率
- ❌ 新折扣率 = 0
- ❌ 没有用户将该游戏加入愿望单
- ❌ 所有用户都关闭了折扣通知

**检查逻辑：**
```java
// 只有满足以下条件才发送邮件
if (discountRate > oldDiscount && discountRate > 0) {
    emailNotificationService.sendDiscountNotification(...);
}
```

## 🛡️ 安全建议

### 生产环境配置

1. **不要硬编码密码**
   ```yaml
   spring:
     mail:
       password: ${MAIL_PASSWORD}  # 从环境变量读取
   ```

2. **使用 .env 文件**
   ```bash
   # .env
   MAIL_PASSWORD=your-smtp-auth-code
   ```

3. **限制API访问**
   - 添加管理员身份验证
   - 使用JWT Token
   - 记录审计日志

4. **邮件限流**
   - 避免短时间内发送大量邮件
   - 考虑使用队列批量发送
