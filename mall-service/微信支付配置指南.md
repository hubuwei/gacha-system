# 微信支付 Native 下单配置指南

## 1. 功能说明

已成功接入微信支付 Native 下单功能，支持生成支付二维码供用户扫码支付。

### 主要接口

1. **Native 下单** - `POST /api/payment/wechat/native/order`
   - 生成微信支付二维码
   
2. **查询订单** - `GET /api/payment/wechat/order/query?orderNo={orderNo}`
   - 查询订单支付状态
   
3. **关闭订单** - `POST /api/payment/wechat/order/close`
   - 关闭未支付的订单
   
4. **支付回调** - `POST /api/payment/wechat/notify`
   - 接收微信支付成功通知

## 2. 配置步骤

### 2.1 申请微信支付商户号

1. 访问 [微信支付商户平台](https://pay.weixin.qq.com/)
2. 注册并申请商户号
3. 完成资质认证

### 2.2 获取必要参数

登录微信支付商户平台，获取以下参数：

- **APPID**: 公众号/小程序的 APPID
- **MCHID**: 商户号
- **API V3 KEY**: APIv3 密钥（在账户中心设置）
- **SERIAL NO**: 商户证书序列号
- **PRIVATE KEY**: 商户私钥文件（apiclient_key.pem）

### 2.3 配置文件修改

编辑 `mall-service/src/main/resources/application.yml`：

```yaml
wechat:
  pay:
    app-id: wx1234567890abcdef        # 替换为你的 APPID
    mch-id: 1234567890                  # 替换为你的商户号
    api-v3-key: your-api-v3-key-here    # 替换为你的 API V3 KEY
    private-key-path: classpath:cert/apiclient_key.pem  # 私钥文件路径
    serial-no: ABCDEF1234567890         # 替换为你的证书序列号
    notify-url: https://your-domain.com/api/payment/wechat/notify  # 回调地址
```

### 2.4 放置证书文件

1. 在 `mall-service/src/main/resources/` 下创建 `cert` 目录
2. 将商户私钥文件 `apiclient_key.pem` 放入该目录

目录结构：
```
mall-service/
└── src/
    └── main/
        └── resources/
            └── cert/
                └── apiclient_key.pem
```

### 2.5 配置回调地址

- `notify-url` 必须是外网可访问的 HTTPS 地址
- 本地开发可使用内网穿透工具（如 ngrok、frp）
- 确保防火墙允许微信服务器访问该地址

## 3. 使用示例

### 3.1 创建订单并获取二维码

```javascript
// 前端调用示例
const response = await fetch('http://localhost:8081/api/orders/create', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    userId: 1,
    paymentMethod: 'wechat',
    items: [
      { gameId: 1, quantity: 1 }
    ]
  })
});

const result = await response.json();

if (result.code === 200) {
  // 获取二维码链接
  const codeUrl = result.data.codeUrl;
  
  // 使用 qrcode.js 生成二维码
  QRCode.toCanvas(document.getElementById('qrcode'), codeUrl);
}
```

### 3.2 轮询查询支付状态

```javascript
// 每 3 秒查询一次订单状态
const checkPaymentStatus = setInterval(async () => {
  const response = await fetch(
    `http://localhost:8081/api/payment/wechat/order/query?orderNo=${orderNo}`
  );
  const result = await response.json();
  
  if (result.data.status === 'SUCCESS') {
    clearInterval(checkPaymentStatus);
    // 支付成功，跳转页面
    window.location.href = `/orders/${orderId}`;
  }
}, 3000);
```

## 5. 常见问题

### Q1: 提示"商户号不存在"
**A:** 检查 `mch-id` 配置是否正确，确保使用的是正式商户号。

### Q2: 提示"签名错误"
**A:** 检查以下配置：
- `api-v3-key` 是否正确
- `private-key-path` 指向的文件是否存在
- `serial-no` 是否与证书匹配

### Q3: 收不到回调通知
**A:** 检查：
- `notify-url` 是否是外网可访问的 HTTPS 地址
- 服务器防火墙是否开放
- 回调接口是否正常响应

### Q4: 本地开发如何测试
**A:** 使用内网穿透工具：
```bash
# 使用 ngrok
ngrok http 8081

# 然后将返回的 https 地址配置为 notify-url
notify-url: https://xxxx.ngrok.io/api/payment/wechat/notify
```

## 6. 安全建议

1. **保护私钥文件**
   - 不要将 `apiclient_key.pem` 提交到 Git
   - 在生产环境使用环境变量或密钥管理服务

2. **验证回调签名**
   - 必须验证微信支付回调的签名
   - 防止伪造的支付通知

3. **使用 HTTPS**
   - 生产环境必须使用 HTTPS
   - 确保 SSL 证书有效

4. **金额校验**
   - 回调时重新计算订单金额
   - 与数据库中的订单金额进行比对
