## ✅ 验证规则

### 用户名规则
- ✓ 长度：3-20 个字符
- ✓ 只能包含：字母（a-zA-Z）、数字（0-9）、下划线（_）
- ✓ 不能为空或只包含空格
- ✓ 不能与已有用户重复

**有效示例：**
- `john_doe`
- `user123`
- `test_user_01`
- `abc`

**无效示例：**
- `ab` (太短)
- `test user` (包含空格)
- `test-user` (包含连字符)
- `@testuser` (包含特殊字符)
- `测试用户` (包含中文)

### 密码规则
- ✓ 长度：6-50 个字符
- ✓ 不能为空
- ✓ 使用 SHA-256 加密存储

**有效示例：**
- `123456`
- `password`
- `MyP@ssw0rd`
- `abcdefghij`

**无效示例：**
- `12345` (太短)
- `` (空密码)

## 💡 小贴士

1. **跨域问题**：已配置全局 CORS，允许所有域名访问
2. **密码安全**：当前使用 SHA-256 加密，生产环境建议使用 BCrypt
3. **Token 机制**：注册后不需要登录即可获取 Token，登录后会返回 JWT Token
4. **初始积分**：每个新用户注册自动赠送 1000 积分
5. **日志查看**：控制台会输出详细的 SQL 和调试信息

## 🔧 故障排查

### 问题 1：服务启动失败

**可能原因：**
- 端口 8081 被占用
- 数据库连接失败

**解决方案：**
```powershell
# 查看端口占用
netstat -ano | findstr :8081

# 修改端口（在 application.yml 中）
server:
  port: 8082
```

### 问题 2：数据库连接失败

**检查项：**
1. MySQL 服务是否启动
2. 数据库 `gacha_system` 是否已创建
3. `application.yml` 中的数据库配置是否正确

### 问题 3：注册时报错"用户名已存在"

**解决方案：**
```sql
-- 清空测试数据（谨慎操作）
DELETE FROM wallet;
DELETE FROM users;

-- 或删除特定用户
DELETE FROM wallet WHERE user_id = 1;
DELETE FROM users WHERE id = 1;
```
