# CMS 运营后台 - 快速使用指南

## 🚀 5分钟快速启动

### 方法一：使用自动化脚本（推荐）

在项目根目录执行：

```powershell
.\start-cms.ps1
```

脚本会自动完成：
- ✅ 检查MySQL连接
- ✅ 初始化数据库表
- ✅ 编译common模块
- ✅ 启动后端服务

然后手动启动前端：

```bash
cd cms-admin
npm run dev
```

### 方法二：手动启动

#### Step 1: 初始化数据库

```bash
mysql -u root -p123456 gacha_system_dev < database/create_cms_tables.sql
```

#### Step 2: 启动后端

```bash
# 编译common模块
cd common
mvn clean install -DskipTests

# 启动CMS服务
cd ../cms-service
mvn spring-boot:run
```

等待看到 "Started CmsServiceApplication" 表示启动成功。

CMS服务将在 `http://localhost:8085` 启动

#### Step 3: 启动前端

打开新终端：

```bash
cd cms-admin
npm run dev
```

#### Step 4: 访问系统

浏览器打开：http://localhost:5174

**登录账号：**
- 用户名：`admin`
- 密码：`admin123`

## 🔍 常见问题排查

### 问题1：后端启动失败

**错误信息：** `Port 8085 was already in use`

**解决方案：**
```bash
# 查找占用端口的进程
netstat -ano | findstr ":8085"

# 终止进程（替换PID为实际进程ID）
taskkill /F /PID <PID>
```

### 问题2：前端无法连接后端

**错误信息：** `Network Error` 或 `CORS error`

**检查清单：**
1. 后端服务是否正常运行
2. `.env` 文件中的API地址是否正确
3. 浏览器控制台是否有CORS错误

**解决方案：**
确保后端Controller有 `@CrossOrigin(origins = "*")` 注解

### 问题3：登录失败

**错误信息：** `用户名或密码错误`

**检查清单：**
1. 数据库中是否存在admin用户
2. 密码是否正确（admin123）
3. 后端日志是否有错误信息

**验证数据库：**
```sql
USE gacha_system_dev;
SELECT username, role, is_active FROM admins WHERE username='admin';
```

### 问题4：数据库表不存在

**错误信息：** `Table 'xxx' doesn't exist`

**解决方案：**
重新执行SQL脚本：
```bash
Get-Content database/create_cms_tables.sql -Encoding UTF8 | mysql -u root -p123456 gacha_system_dev
```
