# 本地开发端口配置说明

## 📋 端口修改概述

为了避开Windows系统的TCP端口排除范围，已将本地开发环境的所有服务端口进行了调整。

---

## 🔧 端口变更对照表

| 服务名称 | 原端口 | 新端口 | 说明 |
|---------|--------|--------|------|
| **mall-service** (商城服务) | 18081 | **17081** | 主商城API服务 |
| **game-service** (游戏服务) | 18082 | **17082** | 游戏相关服务 |
| **gacha-service** (抽卡服务) | 18083 | **17083** | 抽卡系统服务 |
| **auth-service** (认证服务) | 18084 | **17084** | 用户认证服务 |
| **Elasticsearch** (搜索引擎) | 9200 | **17200** | ES搜索服务 |
| **前端Vite开发服务器** | 5173 | 5173 | 未变更 |

---

## ⚠️ Windows端口排除范围（需要避开）

以下是Windows系统保留的端口范围，我们的新端口完全避开了这些范围：

```
7961-8060, 8061-8160, 8161-8260, 8261-8360, 8361-8460
8894-8993, 8994-9093, 9094-9193, 9198-9297, 9298-9397
9428-9527, 9528-9627, 9628-9727, 9728-9827
10248-10347, 10348-10447
12844-12943, 12944-13043
50000-50059 (管理的端口)
```

**新端口选择理由：**
- `17081-17084`：完全在安全范围内，不会被Windows占用
- `17200`：避开9200所在的9198-9297排除范围

## 🚀 本地启动步骤（超详细教程）

### 第一步：启动中间件服务

在启动Java服务之前，需要先启动以下中间件：

#### 1.1 启动MySQL数据库
```powershell
# 如果使用的是本地MySQL，确保服务正在运行
# 检查MySQL是否运行
Get-Service MySQL* | Select-Object Name, Status

# 如果未运行，启动MySQL（服务名可能不同）
Start-Service MySQL80  # 或 MySQL57，根据你的版本
```

#### 1.2 启动Redis
```powershell
# 进入Redis安装目录
cd C:\Program Files\Redis  # 根据你的安装路径调整

# 启动Redis服务器
.\redis-server.exe

# 保持这个窗口打开，不要关闭
```

或者如果你使用Docker：
```powershell
docker run -d --name redis -p 6379:6379 redis:latest
```

#### 1.3 启动RabbitMQ
```powershell
# 方式1：使用Docker（推荐）
docker run -d --name rabbitmq -p 5672:5672 -p 15672:15672 rabbitmq:management

# 方式2：如果本地安装了RabbitMQ
# 在开始菜单找到 "RabbitMQ Service" 并启动
```

访问管理界面：http://localhost:15672 （用户名/密码：guest/guest）

#### 1.4 启动Elasticsearch（重要：使用新端口17200）

⚠️ **注意：这里必须使用17200端口，而不是默认的9200！**

```powershell
# 方式1：使用Docker（推荐）
docker run -d --name elasticsearch -p 17200:9200 -e "discovery.type=single-node" -e "ES_JAVA_OPTS=-Xms512m -Xmx512m" elasticsearch:8.11.0

# 解释：
# -p 17200:9200 表示将容器的9200端口映射到宿主机的17200端口
# 这样你访问 localhost:17200 就能连接到ES
```

验证ES是否启动成功：
```powershell
# 在浏览器访问或使用curl
curl http://localhost:17200

# 应该返回类似这样的JSON：
# {
#   "name" : "...",
#   "cluster_name" : "docker-cluster",
#   "version" : { ... }
# }
```

## 🔍 常见问题排查

### 问题1：端口被占用
```powershell
# 检查端口是否被占用
netstat -ano | findstr "17081"
netstat -ano | findstr "17082"
netstat -ano | findstr "17083"
netstat -ano | findstr "17084"
netstat -ano | findstr "17200"

# 如果被占用，找到对应的PID，然后结束进程
taskkill /F /PID <PID号>
```

### 问题2：Maven启动失败
```powershell
# 确保JAVA_HOME已配置
echo $env:JAVA_HOME

# 确保Maven已安装
mvn -version

# 清理并重新编译
mvn clean install
mvn spring-boot:run
```

### 问题3：数据库连接失败
- 确认MySQL服务正在运行
- 检查数据库名称是否为 `gacha_system_dev`
- 检查用户名密码是否正确（root/123456）

### 问题4：Redis连接失败
- 确认Redis服务正在运行
- 测试连接：`redis-cli ping`，应该返回 `PONG`

### 问题5：Elasticsearch连接失败
- 确认使用了正确的端口17200
- 检查Docker容器是否运行：`docker ps | findstr elasticsearch`
- 查看ES日志：`docker logs elasticsearch`

## 📌 重要提醒

1. **每次启动前**，确保中间件（MySQL、Redis、RabbitMQ、Elasticsearch）已经启动
2. **Elasticsearch必须使用17200端口**，这是通过Docker端口映射实现的
3. **前端会自动代理请求**到正确的后端端口，你只需要访问 http://localhost:5173
4. **如果修改了.env文件**，需要重启对应的Java服务才能生效
5. **建议按顺序启动**：中间件 → 后端服务 → 前端服务

## 🎯 快速启动脚本（可选）

你可以创建一个PowerShell脚本来一键启动所有服务：

```powershell
# start-all-services.ps1

Write-Host "=== 启动所有服务 ===" -ForegroundColor Green

# 启动后端服务（在新窗口中）
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd E:\CFDemo\gacha-system\mall-service; mvn spring-boot:run"
Start-Sleep -Seconds 5

Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd E:\CFDemo\gacha-system\game-service; mvn spring-boot:run"
Start-Sleep -Seconds 5

Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd E:\CFDemo\gacha-system\gacha-service; mvn spring-boot:run"
Start-Sleep -Seconds 5

Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd E:\CFDemo\gacha-system\auth-service; mvn spring-boot:run"
Start-Sleep -Seconds 5

# 启动前端
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd E:\CFDemo\gacha-system\game-mall; npm run dev"

Write-Host "=== 所有服务启动命令已发送 ===" -ForegroundColor Green
Write-Host "请等待各服务启动完成..." -ForegroundColor Yellow
```

使用方法：
```powershell
# 以管理员身份运行PowerShell
.\start-all-services.ps1
```
