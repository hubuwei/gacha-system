# 服务器端口无法启动问题排查指南

## 🔍 问题描述

服务器上的 8081、8082、8083、8084 端口无法正常启动服务。

---

## 📋 诊断步骤

### 步骤 1: SSH 连接到服务器

在 Windows PowerShell 中执行：

```powershell
# 替换为你的实际服务器 IP 地址
ssh root@你的服务器IP地址

# 输入密码后连接成功
```

**示例**：
```powershell
ssh root@123.45.67.89
```

---

### 步骤 2: 检查端口占用情况

连接成功后，执行以下命令：

```bash
# 查看所有正在监听的端口
netstat -tlnp | grep -E '808[1-4]'

# 或者使用 ss 命令（更现代）
ss -tlnp | grep -E '808[1-4]'
```

**可能的结果**：

#### 情况 A: 端口已被占用
```
tcp  0  0 0.0.0.0:8081  0.0.0.0:*  LISTEN  12345/java
tcp  0  0 0.0.0.0:8082  0.0.0.0:*  LISTEN  12346/java
```
说明：端口被其他进程占用，需要查看是什么进程。

#### 情况 B: 端口未被占用（无输出）
说明：端口是空闲的，可以正常使用。

#### 情况 C: 端口被系统保留
说明：可能是 Windows 系统的 Hyper-V 保留了这些端口（但你在 Linux 服务器上，这个可能性较小）。

---

### 步骤 3: 检查是否有残留进程

```bash
# 查找所有 Java 进程
ps aux | grep java

# 或者使用更详细的命令
ps -ef | grep java
```

**查看输出结果**：
- 如果看到有 Java 进程在运行，记录下 PID（进程ID）
- 检查这些进程是否是之前启动的服务

---

### 步骤 4: 检查 PID 文件

从你的截图看到有 `mall-service.pid` 文件，检查它：

```bash
cd /opt/gacha-system

# 查看 PID 文件内容
cat mall-service.pid

# 检查这个进程是否还在运行
ps -p $(cat mall-service.pid)
```

**可能的结果**：

- **进程不存在**：说明服务已停止，但 PID 文件未清理
- **进程存在**：说明服务还在运行，需要停止它

---

### 步骤 5: 检查防火墙设置

```bash
# Ubuntu/Debian 系统
sudo ufw status

# CentOS/RHEL 系统
sudo firewall-cmd --list-all
```

**如果防火墙开启**，需要开放端口：

```bash
# Ubuntu/Debian
sudo ufw allow 8081/tcp
sudo ufw allow 8082/tcp
sudo ufw allow 8083/tcp
sudo ufw allow 8084/tcp

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=8081/tcp
sudo firewall-cmd --permanent --add-port=8082/tcp
sudo firewall-cmd --permanent --add-port=8083/tcp
sudo firewall-cmd --permanent --add-port=8084/tcp
sudo firewall-cmd --reload
```

## 🛠️ 解决方案

### 方案 1: 清理残留进程并重启服务

#### Step 1: 停止所有 Java 进程

```bash
# 方法 A: 杀死特定进程（推荐）
# 先查看进程
ps aux | grep java

# 杀死特定 PID（替换 12345 为实际 PID）
kill -9 12345
kill -9 12346

# 方法 B: 杀死所有 Java 进程（谨慎使用）
pkill -9 java
```

#### Step 2: 清理 PID 文件

```bash
cd /opt/gacha-system

# 删除所有 PID 文件
rm -f *.pid

# 验证已删除
ls -la *.pid
```

#### Step 3: 检查日志文件

```bash
cd /opt/gacha-system/logs

# 查看最新日志
ls -lhrt

# 查看 mall 服务日志
tail -100 mall.log

# 查看启动日志
tail -100 mall-startup.log

# 查看错误日志
tail -100 mall-error.log

# 搜索错误信息
grep -i "error\|exception\|failed" mall.log | tail -20
```

#### Step 4: 检查各服务的配置文件

```bash
# 查看 mall 服务配置
cd /opt/gacha-system/mall
ls -la

# 查看配置文件
cat .env 2>/dev/null || echo "No .env file found"

# 查看 application.yml（如果有）
find . -name "application*.yml" -o -name "application*.properties"
```

#### Step 5: 重新启动服务

```bash
# 进入 mall 服务目录
cd /opt/gacha-system/mall

# 启动服务（根据你的实际启动方式）

# 方式 A: 直接启动
export $(cat .env 2>/dev/null | grep -v '^#' | xargs 2>/dev/null)
nohup java -jar \
  -Dspring.profiles.active=prod \
  -Xms512m \
  -Xmx1024m \
  ../mall-service.jar > ../logs/mall-startup.log 2>&1 &

# 记录 PID
echo $! > ../mall-service.pid

# 查看启动日志
tail -f ../logs/mall-startup.log
```

**等待 30-60 秒**，让服务完全启动。

#### Step 6: 验证服务

```bash
# 检查进程
ps aux | grep java

# 检查端口
netstat -tlnp | grep 8081

# 健康检查
curl http://localhost:8081/api/actuator/health

# 如果看到 {"status":"UP"} 说明启动成功
```

### 方案 2: 检查端口是否被系统保留（Linux）

虽然在 Linux 上较少见，但仍需检查：

```bash
# 检查端口范围

sudo netstat -anp | grep -E ':(8081|8082|8083|8084)'
```

### 方案 4: 查看详细错误日志

如果服务启动失败，查看详细的错误信息：

```bash
cd /opt/gacha-system/logs

# 查看最近的启动日志
tail -200 mall-startup.log

# 搜索特定错误
grep -i "bind\|address\|port\|permission\|denied" mall-startup.log

# 查看最后 50 行错误
tail -50 mall-error.log

# 实时查看日志（启动服务时在另一个终端执行）
tail -f mall-startup.log
```

**常见错误及解决方案**：

#### 错误 1: Address already in use
```
java.net.BindException: Address already in use
```
**解决**：端口被占用，参考方案 1 清理进程。

#### 错误 2: Permission denied
```
java.net.BindException: Permission denied
```
**解决**：
```bash
# 普通用户无法绑定 1024 以下端口，但 8081 应该没问题
# 检查是否有 SELinux 限制
sudo getenforce

# 如果是 Enforcing，临时关闭测试
sudo setenforce 0
```

#### 错误 3: Connection refused（数据库）
```
java.sql.SQLException: Access denied for user
```
**解决**：检查数据库配置和密码。

#### 错误 4: Cannot connect to Redis
```
org.springframework.data.redis.RedisConnectionFailureException
```
**解决**：确保 Redis 正在运行且密码正确。

#### 错误 5: Out of Memory
```
java.lang.OutOfMemoryError: Java heap space
```
**解决**：增加 JVM 内存
```bash
# 修改启动命令中的参数
-Xms1024m -Xmx2048m
```

### 方案 5: 使用 Systemd 管理（推荐）

如果你还没有配置 Systemd 服务，建议使用它来管理：

#### Step 1: 创建服务文件

```bash
sudo vim /etc/systemd/system/mall-service.service
```

添加以下内容：

```ini
[Unit]
Description=Mall Service - Game Shopping Platform
After=network.target mysql.service redis.service rabbitmq-server.service elasticsearch.service
Wants=mysql.service redis.service rabbitmq-server.service elasticsearch.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/gacha-system/mall

# 如果有 .env 文件
EnvironmentFile=/opt/gacha-system/mall/.env

# 启动命令（根据你的实际路径调整）
ExecStart=/usr/bin/java \
    -Xms512m \
    -Xmx1024m \
    -Dspring.profiles.active=prod \
    -jar /opt/gacha-system/mall-service.jar

# 重启策略
Restart=on-failure
RestartSec=10
StartLimitIntervalSec=60
StartLimitBurst=3

# 日志
StandardOutput=append:/opt/gacha-system/logs/mall-startup.log
StandardError=append:/opt/gacha-system/logs/mall-error.log

# 安全限制
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
```

#### Step 2: 启用并启动服务

```bash
# 重新加载 systemd 配置
sudo systemctl daemon-reload

# 启用开机自启
sudo systemctl enable mall-service

# 启动服务
sudo systemctl start mall-service

# 查看状态
sudo systemctl status mall-service

# 查看实时日志
sudo journalctl -u mall-service -f
```

#### Step 3: 常用管理命令

```bash
# 停止服务
sudo systemctl stop mall-service

# 重启服务
sudo systemctl restart mall-service

# 查看状态
sudo systemctl status mall-service

# 查看日志（最近 100 行）
sudo journalctl -u mall-service -n 100

# 查看错误日志
sudo journalctl -u mall-service -p err

# 查看今天的日志
sudo journalctl -u mall-service --since today
```

## 🎯 完整的重启流程（一步一步来）

### 1. 连接到服务器

```powershell
ssh root@你的服务器IP
```

### 2. 停止所有服务

```bash

pkill -9 java

```

### 3. 检查依赖服务

```bash

sudo systemctl status redis | head -5

sudo systemctl status rabbitmq-server | head -5
```

**如果有服务未运行，启动它们**：
```bash
sudo systemctl start mysql redis rabbitmq-server elasticsearch
```

### 4. 检查端口

```bash
# 确认端口是空闲的
netstat -tlnp | grep -E '808[1-4]'

# 应该没有输出，说明端口空闲
```

### 5. 启动 Mall Service

```bash
cd /opt/gacha-system/mall

# 如果有 .env 文件，加载环境变量
if [ -f .env ]; then
    export $(cat .env | grep -v '^#' | xargs)

  -Dspring.profiles.active=prod \
  -Xms512m \

```



## 📝 快速诊断脚本

创建一个脚本来自动诊断：

```bash
# 创建诊断脚本
cat > /opt/gacha-system/diagnose.sh << 'EOF'
#!/bin/bash

echo "======================================"
echo "  Gacha System 诊断脚本"
echo "======================================"
echo ""

echo "1. 检查 Java 进程"
echo "--------------------------------------"
ps aux | grep java | grep -v grep
echo ""

echo "2. 检查端口占用"
echo "--------------------------------------"
for port in 8081 8082 8083 8084; do
    result=$(netstat -tlnp | grep ":$port " 2>/dev/null)
    if [ -n "$result" ]; then
        echo "端口 $port 被占用: $result"
    else
        echo "端口 $port 空闲"
    fi
done
echo ""

echo "3. 检查依赖服务"
echo "--------------------------------------"
services=("mysql" "redis" "rabbitmq-server" "elasticsearch")
for service in "${services[@]}"; do
    if systemctl is-active --quiet "$service"; then
        echo "✓ $service 运行中"
    else
        echo "✗ $service 未运行"
    fi
done
echo ""

echo "4. 检查日志文件"
echo "--------------------------------------"
ls -lh /opt/gacha-system/logs/ 2>/dev/null || echo "日志目录不存在"
echo ""

echo "5. 检查 PID 文件"
echo "--------------------------------------"
ls -lh /opt/gacha-system/*.pid 2>/dev/null || echo "无 PID 文件"
echo ""

echo "6. 检查磁盘空间"
echo "--------------------------------------"
df -h /opt/gacha-system
echo ""

echo "7. 检查内存使用"
echo "--------------------------------------"
free -h
echo ""

echo "诊断完成！"
EOF

# 赋予执行权限
chmod +x /opt/gacha-system/diagnose.sh

# 运行诊断
/opt/gacha-system/diagnose.sh
```
